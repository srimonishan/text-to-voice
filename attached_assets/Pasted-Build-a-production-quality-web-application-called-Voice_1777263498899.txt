Build a production-quality web application called “VoiceForge AI” using Node.js (Express) for the backend and vanilla HTML, CSS, and JavaScript for the frontend (no frameworks required).

The application must integrate the ElevenLabs Text-to-Speech API to convert user input text into high-quality voice output in real time.

CORE CONSTRAINTS:
- No database (no Supabase, Firebase, MongoDB, or any storage)
- Stateless architecture only
- Fully functional end-to-end
- Clean, modern, minimal UI
- Ready to run in Replit without modification

FEATURES:

1. TEXT INPUT
- Large textarea for user input
- Character counter
- Placeholder text guiding usage

2. VOICE SETTINGS
- Dropdown with at least 3 predefined ElevenLabs voice IDs
- Sliders for:
  - Stability (0–1)
  - Similarity Boost (0–1)

3. CONTENT MODE SELECTOR
- Buttons:
  - Motivational
  - Storytelling
  - Educational
- Each mode slightly modifies tone of text before sending to API

4. GENERATE VOICE
- Button triggers API call
- Disable button during processing
- Show loading spinner

5. AUDIO OUTPUT
- Built-in audio player
- Play / Pause controls
- Download MP3 button

BACKEND REQUIREMENTS (Node.js + Express):

- Create POST endpoint: /generate-voice

- Call ElevenLabs API:
  https://api.elevenlabs.io/v1/text-to-speech/{voice_id}

- Headers:
  xi-api-key: process.env.ELEVENLABS_API_KEY
  Content-Type: application/json

- Request body:
{
  "text": "<processed_text>",
  "voice_settings": {
    "stability": <value>,
    "similarity_boost": <value>
  }
}

- Return:
  Audio buffer (MP3) directly to frontend
